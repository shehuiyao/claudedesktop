你想要的"切换模型"是哪种？

当前应用已支持切换 Claude / Gemini / Codex / 火山，这次想扩展到哪个层次？

A

更多 CLI 工具包装

继续沿用现有架构（PTY 包装 CLI），接入更多命令行 AI 工具，例如 Kimi、通义千问 CLI、Cursor CLI 等。

优点

  • 改动小，风险低
  • 与现有架构一致

限制

  • 依赖各厂商提供 CLI 工具
  • 体验受限于 CLI 本身
B

会话内切换具体模型版本

在新建 Tab 时，可以选择具体模型版本：比如 claude-opus-4 / claude-sonnet-4.5 / claude-haiku，或 codex-mini / codex-full。通过 CLI 参数 --model 传递。

优点

  • 精细控制模型版本
  • 无需改变整体架构

限制

  • 仅限 CLI 支持的模型
C

直接 API 调用（像 OpenClaw 一样）

新增一种"API 模式"，绕过 CLI，直接调用各大厂商 API（Anthropic、OpenAI、Gemini、火山、通义等），自由选择任何模型。用自己的 API Key，低延迟，无 CLI 依赖。

优点

  • 模型选择自由度最高
  • 不依赖第三方 CLI
  • 可精细控制 system prompt、temperature 等

限制

  • 需要较大改动（Rust 后端新增 HTTP 调用）
  • 需自行管理各厂商 API Key
  • 失去 Claude Code / Codex 的工程化功能（工具调用、文件操作等）

也可以多选——例如同时要 B + C
